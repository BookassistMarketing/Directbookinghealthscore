# Handoff — 18 June 2026

Session on **Direct Booking Health Score**. Brainstormed and spec-approved a hidden `/staff` bypass page for the lead-capture gate on both audits. No code written yet — design doc committed, ready for implementation planning next session.

## What we're building

A hidden `/staff` route at `directbookinghealthscore.com/staff`. Bookassist staff enter a shared password, then click through to either audit and skip the HubSpot lead-capture gate entirely. No CRM pollution, no per-user accounts, no admin dashboard — just a soft gate that lets internal users demo and test cleanly.

## Decisions locked in

| Question | Answer |
|---|---|
| Auth model | **Single shared password** (env var `STAFF_PASSWORD`) |
| Entry flow | **Dedicated `/staff` hub page** — password form, then two buttons (Hotel Tech Audit / AI Visibility Audit) |
| Form fields | **Password only** — no email, no per-user identity |
| Scope | **Both audits** — Hotel Tech Audit AND AI Visibility Audit |
| HubSpot logging | **No** — staff bypass leaves no trace in HubSpot |
| Password verification | **Server-side** — `/api/staff/auth` route. Password never enters JS bundle. |
| Session token | **HMAC-signed**, payload `{iat, exp}`, signed with `STAFF_TOKEN_SECRET` |
| Session lifetime | **30 days** in `localStorage` |
| Localisation | **English only** — `/staff` not translated. Other locales 404. |
| Sitemap / SEO | **Hidden** — not in sitemap, `noindex,nofollow` meta tag |
| Nav link | **None** — URL shared internally only |

## Architecture summary

- **`app/staff/page.tsx`** — password form + signed-in hub view
- **`app/api/staff/auth/route.ts`** — `POST` with password → returns HMAC-signed token. Constant-time compare, 200-500ms random delay on failure, in-memory rate limit (10 / 15 min / IP).
- **`app/api/staff/verify/route.ts`** — `POST` with token → returns `{valid: true}` or 401. Used by audit pages on mount to confirm bypass.
- **`lib/staffBypass.ts`** — shared `checkStaffBypass()` helper; reads localStorage token, calls verify endpoint.
- **`components/AiAudit.tsx`** + **`components/AuditTool.tsx`** — modified to call `checkStaffBypass()` on mount; when true, skip the `LeadCapture` modal entirely (parent just doesn't mount it). Adds a "Bookassist Staff Mode" pill badge.
- **`components/LeadCapture.tsx`** — **unchanged**.

## Env vars to add (Amplify + `.env.local`)

- `STAFF_PASSWORD` — the shared password
- `STAFF_TOKEN_SECRET` — 32 random hex bytes (`openssl rand -hex 32`), used to sign tokens

Neither uses `NEXT_PUBLIC_` prefix. Both are server-side only.

## Threat model

Soft gate. The aim is to keep bots and casual visitors out, not to defend against motivated attackers. If the password leaks → rotate `STAFF_PASSWORD`. To force-logout every existing browser → also rotate `STAFF_TOKEN_SECRET`. If real auth ever becomes needed (e.g. customer data on the staff side), migrate to Cognito.
